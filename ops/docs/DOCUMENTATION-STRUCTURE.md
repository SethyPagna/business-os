# Documentation Structure

This project uses a layered documentation model to keep architecture, operations, and code-reference details organized without changing runtime behavior.

## 1. Product-Level Docs

- `README.md`
  - user-facing project overview and quick-start instructions.
- `README-build.md`
  - release packaging and deployment workflow.

## 2. Architecture and Ownership Docs (`docs/`)

- `ARCHITECTURE.md`
  - high-level backend/frontend/runtime design.
- `SCHEMA-RELATIONSHIPS.md`
  - data model relationships and key dependencies.
- `BACKEND-ROUTES.md`
  - API route inventory by domain.
- `FILE-MAP.md`
  - backend/frontend ownership map.
- `DOCUMENTATION-STRUCTURE.md`
  - this document.

## 3. Auto-Generated Reference Docs (`docs/reference/`)

- `BACKEND-FUNCTION-REFERENCE.md`
  - backend file-level symbol inventory.
- `FRONTEND-FUNCTION-REFERENCE.md`
  - frontend file-level symbol inventory.
- `TRANSLATION-REFERENCE.md`
  - i18n key coverage and key-diff report.
- `RUN-RELEASE-REFERENCE.md`
  - setup/start/stop/build script inventory.
- `MODULE-NAMING-GUIDE.md`
  - modular naming standard for safe incremental refactor.
- `PROJECT-COVERAGE-SUMMARY.md`
  - all-files/all-folders first-party coverage summary.
- `ALL-FILE-INVENTORY.md`
  - per-file commentary for all frontend/backend first-party files.
- `FOLDER-COVERAGE.md`
  - per-folder commentary and contained-file listing.
- `IMPORT-EXPORT-REFERENCE.md`
  - import/export and dependency cross-reference map.
- `TRANSLATION-SECTION-REFERENCE.md`
  - translation keys grouped by page/domain sections.
- `PERFORMANCE-SCAN.md`
  - source/chunk size and hotspot scan output.

Generated by:

```bash
node scripts/generate-doc-reference.js
node scripts/generate-full-project-docs.js
node scripts/performance-scan.js
```

## 4. Folder-Level READMEs

- `backend/src/README.md`
- `backend/src/routes/README.md`
- `backend/src/services/README.md`
- `frontend/src/README.md`
- `frontend/src/components/README.md`

These explain localized module responsibilities for faster onboarding.
